# Docker Compose Version Command Lab
## Overview
The Docker Compose version command is used to display the Docker Compose version information. This lab will guide you through the process of checking the Docker Compose version using the `docker-compose version` command.

## Prerequisites
Before starting this lab, ensure you have:
* Created an account with DockerHub
* Opened the Play with Docker (PWD) Platform on your browser
* Added a new instance of Alpine OS on the PWD Platform

## Step-by-Step Instructions
To check the Docker Compose version, follow these steps:
1. Open the terminal in your Alpine OS instance on the PWD Platform.
2. Run the command to check the Docker Compose version:
```bash
$ docker-compose version
```
3. Observe the output, which will display the Docker Compose version information.

## Code Examples and Expected Output
The `docker-compose version` command will display the following information:
```bash
docker-compose version 1.23.2, build 1110ad0
docker-py version: 3.7.3
CPython version: 2.7.16
OpenSSL version: OpenSSL 1.1.1c
28 May 2019
```
Note that `docker-py` is the Docker Remote API, which allows you to manage Docker containers and images from within Python.

## Key Concepts and Learning Objectives
This lab covers the following key concepts:
* Using the `docker-compose version` command to display Docker Compose version information
* Understanding the role of `docker-py` in managing Docker containers and images from within Python
* Familiarity with the Play with Docker (PWD) Platform and Alpine OS instance

By completing this lab, you will gain hands-on experience with the Docker Compose version command and understand its importance in managing Docker containers and images.
